
| Stav        | Poslat notifikaci? | Důvod                                           |
| ----------- | ------------------ | ----------------------------------------------- |
| `Created`   | ✅ Ano              | Potvrzení o vytvoření objednávky                |
| `Paid`      | ✅ Ano              | Informace o úspěšné platbě                      |
| `Accepted`  | ✅ Ano              | Uživatel ví, že objednávka byla schválena       |
| `Rejected`  | ✅ Ano              | Kritická – musí vědět, že objednávka padla      |
| `Shipped`   | ✅ Ano              | Odesláno – důležité pro sledování               |
| `Completed` | ⚠️ Možná           | Může být notifikace typu: "Děkujeme za nákup!"  |
| `Cancelled` | ✅ Ano              | Pokud zrušil uživatel sám, měl by mít potvrzení |





| Stav         | Poslat notifikaci? | Důvod                                             |
| ------------ | ------------------ | ------------------------------------------------- |
| `Pending`    | ❌ Ne               | Uživatel to často neřeší, defaultní stav          |
| `InProgress` | ⚠️ Volitelné       | Může být užitečné: „Vaše zásilka je na cestě“     |
| `Delivered`  | ✅ Ano              | Důležitá – uživatel chce vědět, že zboží dorazilo |
| `Canceled`   | ✅ Ano              | Kritická – musí být informován, že něco padlo     |




 Nové notifikace (např. OrderStatusChanged)
Vytvoříš nový Event → OrderStatusChangedEvent.

Pošleš ho v metodě ChangeOrderStatusAsync v OrderService.

V NotificationService.Api.Consumers vytvoříš nový OrderStatusChangedConsumer.

V něm vytvoříš notifikaci, uložíš ji do DB a pošleš přes SignalR.
